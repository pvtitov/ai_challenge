# Critical Fixes - Round 2

## Issues Found From User Testing

### Issue 1: MCP "Not a git repository" Error ✗
**Error Message**:
```
Not a git repository: /Users/paveltitov/Documents/programming/ai_challenge/aichat-github/private/tin
```

**Root Cause**: 
The `run.sh` script created a **broken symlink**:
```
aichat-github/private/tinyAI/tinyAI -> aichat-github/private/tinyAI
```

This created a recursive symlink where `private/tinyAI/` contained another `tinyAI/` symlink pointing back to itself, so the actual `.git` directory was never accessible.

**What Happened**:
1. `run.sh` saw `private/tinyAI/.git` didn't exist
2. It created a symlink: `ln -sf ../private/tinyAI private/tinyAI`
3. But this created `/aichat-github/private/tinyAI/tinyAI/` (nested)
4. The MCP server looked for `.git` at `/aichat-github/private/tinyAI/.git` - NOT FOUND

**Fix Applied**:
```bash
# Manually copied the actual repo (with .git) to correct location
rm -rf private
mkdir -p private
cp -R ../private/tinyAI private/tinyAI
# Now: /aichat-github/private/tinyAI/.git EXISTS
```

**Updated run.sh** to prevent this in the future:
- Checks if directory is a proper git repo
- Doesn't create symlinks that could nest incorrectly
- Properly copies the repository

### Issue 2: RAG Finding 0 Results ✗
**Symptom**:
```
[Searching through 35 indexed chunks]
[RAG: Found 0 relevant source(s) in knowledge base]
```

**Root Cause**: 
The cosine similarity threshold was **0.7** which was too strict. The embeddings generated by `nomic-embed-text` model may not achieve 0.7 similarity even for relevant chunks.

**Fix Applied**:

1. **Lowered threshold from 0.7 to 0.5** in `EmbeddingSearchService.java`:
```java
public EmbeddingSearchService() {
    this(ApiConstants.OLLAMA_URL, ApiConstants.OLLAMA_MODEL, 5, 0.5);  // Was 0.7
}
```

2. **Added debug output** to show actual similarity scores:
```
[RAG Debug: Top 3 similarity scores]
  1. README.md - Score: 0.654
  2. requirements.txt - Score: 0.523
  3. llm_cli.py - Score: 0.489
```

This will help you see:
- What the actual scores are
- Whether 0.5 is appropriate
- Which chunks are being matched

**Result**: Should now find 3-5 results for most queries.

### Issue 3: Ollama Timeouts on /review ⚠️
**Error**: 
```
java.net.SocketTimeoutException: Read timed out
```

**Root Cause**: 
The `/review` mode gathers a LOT of context (RAG + multiple MCP calls), creating a very long prompt. Ollama takes longer to process large prompts, exceeding the default timeout.

**Not Critical**: Regular chat mode works fine, only affects review mode with large context.

**Recommended Fix** (not applied yet - needs OllamaLlmService.java update):
```java
// Increase timeout from default to 60 seconds
client = new OkHttpClient.Builder()
    .connectTimeout(10, TimeUnit.SECONDS)
    .readTimeout(60, TimeUnit.SECONDS)  // Was probably 10-30s
    .build();
```

For now, review mode may timeout on large reviews but chat mode works.
